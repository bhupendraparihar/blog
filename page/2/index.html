looping problem

I was looking into a problem of closure inside loop in stackoverflow. It is as follows..

var funcs = {};
for (var i = 0; i & lt; 3; i++) { // let's create 3 functions
    funcs[i] = function () { // and store them in funcs
        console.log("My value: " + i); // each should log its value.
    };
}
for (var j = 0; j & lt; 3; j++) {
    funcs[j](); // and now let's run each one to see
}

the output I was expecting is 

My value: 0
My value: 1
My value: 2

But the output is much different.

My value: 3
My value: 3
My value: 3

Now the reason for such an awkward behaviour is within each of our anonymous functions, is bound to the same variable outside of the function.

This problem can be solved by closure.

var funcs = {};
for (var i = 0; i < 3; i++) {          // let's create 3 functions
  funcs[i] = (function(i){
    //console.log(i)

    return function() {            // and store them in funcs
       console.log("My value: " + i); // each should log its value.
    };
  })(i);
}
for (var j = 0; j < 3; j++) {
    funcs[j]();                        // and now let's run each one to see1
}

or just by using let instead of var, as let has block level scope.

var funcs = {};
for (let i = 0; i & lt; 3; i++) { // let's create 3 functions
    funcs[i] = function () { // and store them in funcs
        console.log("My value: " + i); // each should log its value.
    };
}
for (var j = 0; j & lt; 3; j++) {
    funcs[j](); // and now let's run each one to see
}

truthy and falsy

In JavaScript, apart from Boolean values like true and false, all other values also have some boolean value corresponds to it.

    false
    ""(empty string)
    0(zero)
    undefined
    null
    NaN(not a number)

are considered falsy values.

So all the other values except these are, truthy values.

Ex “false” (String false) under conditional statement returns true.

This is very important concept in JavaScript.

public and private

Here is the code for creating a object definition(class) which maintains the public and private scope of variable.

function Person(new_name){
    //_age is a private variable
    var _age = "";

    var obj = {};

    obj.name = new_name;
    obj.getAge = function() {
        return _age;
    }
    obj.setAge = function (new_age) {
        _age = new_age;
    }
    obj.displayInfo = function () {
        return "My age is " + _age;
    }
    return obj;
}

var person = new Person('BP');
person.setAge(26);
console.log(person.getAge()); //26
console.log(person._age); //undefined

You can observe, it is closure what keeps the varible private
